  • Installing Pylint
    • The shell depends on what operating system you use
    • But wherever you installed python, pip should be installed
    • To call pip, you run python3 -m pip
    • To get Pylint, you run python3 -m pip install pylint
    • If you use WSL and a debian-based OS, you need to run sudo apt install python3-pip to install pip
    • Then run pip install pylint
    • If all else fails, follow this link: PIP Install
  • Running Pylint
    • In your shell, you go to the directory where your code is
    • Run pylint file path or file name
    • And that's it!
  • OOP Terms and Definitons
    • Factory - an object for creating other objects – formally a factory is a function or method that returns objects of a varying prototype or class from some method call
    • Instantiation - creation of an object
    • Object - An object (instance) is an instantiation of a class. When class is defined, only the description for the object is defined. Therefore, no memory or storage is allocated.
    • Class - blueprint for an object. Defines everything about an object
    • Namespace - collection of currently defined symbolic names along with information about the object that each name references:
      • Built-In Global Enclosing Local
    • Constructor - It prepares the new object for use, often accepting arguments that the constructor uses to set required member variables
    • Fixture - Only in tests: ensure that there is a well known and fixed environment in which tests are run so that results are repeatable
    • Unit Test - testing method by which individual units of source code—sets of one or more computer program modules together with associated control data, usage procedures, and operating procedures—are tested to determine whether they are fit for use
    • Static Method - Static functions are helpful as they do not rely on an instantiated member of whatever class they are attached to
    • Class Method bound to a class rather than an object. Class methods can be called by both class and object. These methods can be called with a class or with an object. takes cls as the first parameter can access or modify the class state
    • Facade - Providing simple Interface: One of the most important application of Facade Method is that it is used whenever you want to provide the simple interface to the complex sub-system Division into layers: It is used when we want to provide a unique structure to a sub-system by dividing them into layers. It also leads to loose coupling between the clients and the subsystem. Masks actual functions into one line functions to hide what the code actual does
    • Design Pattern Used - general repeatable solution to a commonly occurring problem in software design
    • Method - functions associated with the objects of the class they belong to
    • Property - provides an interface to instance attributes. It encapsulates instance attributes and provides a property, takes the get, set and delete methods as arguments and returns an object of the property class
