Example of Abstract Class and Polymorphism

  • Main Principles of Object Oriented Programming
    • Abstraction
      • Abstract classes are blueprints for other classes
      • This is demonstated by the Calculation class
      • All this class does is takes in a tuple, converts it to a list of floats, and saves it as an attribute
      • By itself, this class does nothing because there is no implementation in any methods
      • And this is the main criteria of an abstract class
    • Polymorphism
      • This is basically using the same function but with different types of data
      • An example of this in python is len(), because you can use that function on most data types such as int, list, and dict
      • This is showcased with the get_result() function in the addition, subtraction, division, and multiplication classes
      • As long as you call get_result(), you will always get the result no matter what is in Calculator.history

Example of Encapsulation and Inheritence

  • Main Principles of Object Oriented Programming
    • Encapsulation
      • This is the technique of masking functions or attributes by having them only be called through other functions
      • This is showcased in add_numbers(), subtract_numbers(), multiply_numbers(), and divide_numbers() in the Calculator class
      • All four of those methods call add_calculation() in the Calculator class and create() in the Calculation class
      • By calling create(), the above four methods also receive access to the get_result() method
      • Another way of looking at this is:
        get_result() is in create() which is in add_numbers() along with add_calculation()
      • And these nested functions is encapsulation
    • Inheritence
      • Inheritence is the functions and attributes that get imported and utilized in other classes
      • The addition, subtraction, division, and multiplication classes all import the calculation class
      • This means they have access to the functions and attributes in the calculation class
      • The create() is inherited because it is not defined in the four operation classes
      • Also, self.values is not defined in the operation classes, which means it was inherited from the calcuation class
